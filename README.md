# 小番茄 · 视频+图片合并版

> 一个基于 Web 的在线视频解密 + 图片加密/解密工具，合并自独立的视频解密工具和图片加解密工具。支持三种核心混淆算法，全浏览器端运行，无需服务器。
>
> **注意**：视频模块仅支持解密（实时预览），不支持视频加密；图片模块同时支持加密和解密。

## ✨ 功能概览

### 🎬 视频解密（实时预览）

- 支持 **吉尔伯特曲线** 和 **分块打乱** 两种解密方式
- 基于 **WebGL2** 的 GPU 加速实时渲染
- 全屏播放，支持双指缩放与滑动快进/快退
- 可拖动进度条，实时反馈解密进度
- 移动端适配，支持横屏/竖屏显示

### 🖼️ 图片加密/解密

支持三种核心混淆算法，用户可在同一页面自由切换，无需跳转其他网站或小程序：

#### 1️⃣ 吉尔伯特曲线（Hilbert Curve）
- 基于空间填充曲线的像素级置乱算法
- **WASM 深度优化**，WebAssembly 大幅提升批量处理性能
- 大部分设备均可流畅运行，支持 WASM Worker 多线程加速
- 支持密钥（Key），可重置加密偏移量

#### 2️⃣ 分块打乱（Block Shuffle）
- 将图片划分为自定义大小的块，按密钥随机打乱顺序
- 块大小可自由配置（像素级），支持标准化（高度=宽度）
- 多文件模式下自动降级为像素级打乱

#### 3️⃣ 最速混淆 / 像素置乱（Fastest Obfuscation）
- 基于伪随机数生成器的像素行置乱算法
- 算法直接移植自小程序，保证 **100% 兼容性**
- 极快处理速度，适合大批量图片处理
- 固定密钥设计，与小程序默认密钥一致

## ⚡ 性能优化

本站相比传统网站做了大量深度优化：

### 🧠 多级加速管线

```
WASM Worker → Main Thread WASM → WebGL GPU → Worker CPU → Main Thread CPU
```

系统会根据设备能力自动选择最优路径，确保在各类设备上都能获得最佳性能。

### 🔧 具体优化措施

| 优化项 | 说明 |
|--------|------|
| **WASM 加速** | 吉尔伯特曲线和分块打乱算法编译为 WebAssembly，批量处理优势明显 |
| **Web Worker** | 像素重映射在独立线程中执行，不阻塞主线程 UI |
| **WebGL GPU 重映射** | 大图自动启用 GPU 加速，利用着色器并行处理 |
| **映射缓存** | 相同尺寸和密钥的像素映射表自动缓存，重复加解密无需重新计算 |
| **分片 yield** | 长任务分片执行，定期让出主线程，保持页面响应 |
| **移动端保活** | Wake Lock API + 静音音频 + 模拟触摸事件，防止移动端浏览器休眠中断处理 |

### 📱 设备兼容

- **高性能设备**：启用完整 WASM + Worker + GPU 管线
- **中低端设备**：自动降级到 JS + CPU 模式，兼容绝大多数浏览器
- **极低端设备**：兼容模式运行，保证功能可用

## 🔐 额外功能

### 色彩反转（XOR 二次加密）
- 基于 XOR 的像素色彩反转，作为第二层加密
- 需要 WEBP 编码支持（自动检测）
- 可在加密/解密时与其他算法叠加使用

### 大图提醒
- 导入超过 1080×1920 像素的大图时自动弹窗提醒
- 用户可选择 **压缩图片**（按比例缩小分辨率）或 **保持原图**
- 适合发布混淆图前压缩，降低平台二次压缩风险
- 可选择"以后不再提醒"

### 批量处理
- 支持对多张图片批量加解密
- 可指定处理次数（重复混淆）
- 进度条实时反馈

### 加密计数器
- 记录加密/解密操作次数
- **长按计数器**可一键调出批量处理弹窗，方便快速设置混淆次数

### 导出方式
| 方式 | 说明 |
|------|------|
| **直接下载** | 逐个下载单张图片 |
| **打包下载** | 所有图片打包为 ZIP 文件下载 |
| **导出至文件夹** | 使用 File System Access API 直接保存到系统目录 |
| **导出密钥文件** | 可选的 .txt 文件，记录加密参数、密钥、处理次数等 |

### 输出格式
- JPG (95% 质量)
- PNG（无损）
- WebP（无损，需浏览器支持）

### 图片预览
- 支持左右切换浏览处理前后的图片
- 支持全屏查看 + 双指缩放
- 点击计数器可关闭预览

### 重加密模式
- 开启后允许对已加密的图片再次加密
- 关闭后每次导入新图片自动重置状态，避免误操作

## 🚀 部署

直接上传到 Web 服务器根目录即可。

### 文件结构

```
├── index.html                  # 主页面
├── tb.ico                      # 网站图标
├── README.md                   # 本说明文档
├── LICENSE                     # 开源协议
├── .gitignore                  # Git 忽略规则
├── css/
│   └── style.css               # 样式文件
├── js/
│   ├── app.js                  # 主逻辑（视频+图片模块）
│   ├── fastest-obfuscation.js  # 最速混淆算法
│   ├── image-wasm-worker.js    # WASM Worker
│   ├── jszip.min.js            # ZIP 打包库
│   └── FileSaver.min.js        # 文件保存库
└── wasm/
    ├── hilbert_image_cipher_wasm_bg.wasm
    └── hilbert_image_cipher_wasm.js
```

### 要求
- 必须通过 HTTP/HTTPS 访问（`file://` 协议无法加载 WASM）
- 服务器需正确返回 `.wasm` 文件的 MIME 类型：`application/wasm`

### 本地测试

```powershell
python -m http.server 8125 --bind 0.0.0.0
```

访问 `http://localhost:8125/`
